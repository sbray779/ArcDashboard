# Arc Dashboard

An Azure Monitor Workbook deployed via Terraform that surfaces Azure Arc-enabled server inventory and SQL Server extension health across one or more subscriptions using Azure Resource Graph.

---

## What It Does

The workbook provides two tables on a single page, both populated at load time via live Azure Resource Graph queries:

### Arc Servers WITH SQL Extension
Servers that have either `WindowsAgent.SqlServer` or `LinuxAgent.SqlServer` installed. Columns include:

| Column | Source |
|---|---|
| Server Name / Display Name / Computer Name | Arc machine resource |
| OS Name / Type / Edition / SKU | Arc machine properties |
| Server State / Status | Arc agent provisioning and connectivity state |
| Total Memory (GB) / Processor / Cores / Logical Cores | Arc detected properties |
| MSSQL Discovered | Whether SQL was auto-detected on the machine |
| ESU License / ESU Eligibility | **Windows Server ESU** — from `licenseProfile.esuProfile` on the Arc machine |
| Server Type | Standard / Datacenter (from ESU profile) |
| Extension Name / Version / State | SQL Server extension metadata |
| Extension License | SQL license type configured on the extension |
| SQLExtension Status | Health of the SQL Server Extension Agent |
| DPS Status | Upload status to the Data Processing Service (feeds SQL inventory, BPA, Defender) |
| BPA Enabled | Whether Best Practices Assessment is configured |
| ESU Enabled | **SQL Server ESU** — from `enableExtendedSecurityUpdates` on the extension |
| Subscription Id / Location / Resource Group / Tags | Resource metadata |

> **Note:** ESU License/Eligibility (Windows Server ESU) and ESU Enabled (SQL Server ESU) are independent programs.

### Arc Servers WITHOUT SQL Extension
Servers with no `WindowsAgent.SqlServer` or `LinuxAgent.SqlServer` extension present. Uses a `leftouter` join filtered by `isempty()` to exclude matched servers. Columns are the same as above minus the extension-specific fields.

## Repository Structure

```
ArcDashboard/
├── providers.tf              # AzureRM (~3.90) and Random provider configuration
├── variables.tf              # Input variables: resource group, location, display name, tags
├── main.tf                   # Resource group, random UUID, and workbook resource
├── outputs.tf                # Workbook resource ID and direct portal URL
├── terraform.tfvars.example  # Template — copy to terraform.tfvars and customize
├── workbook/
│   └── arc-dashboard.json    # Azure Monitor Workbook definition (Notebook/1.0 schema)
└── .gitignore                # Excludes state files, provider binaries, plan files, tfvars
```

### Key files

**`main.tf`**
Deploys two Azure resources:
- `azurerm_resource_group` — resource group to contain the workbook
- `azurerm_application_insights_workbook` — the standalone workbook (no Application Insights dependency required); the name is a random UUID generated by `random_uuid`

**`workbook/arc-dashboard.json`**
The workbook definition in Azure Monitor Notebook 1.0 format. Uses:
- `"queryType": 1` — Azure Resource Graph query type
- `"resourceType": "microsoft.resourcegraph/resources"` — required exact value for ARG queries
- `"crossComponentResources": ["value::all"]` — fans the query out across all subscriptions the viewer has access to
- `"fallbackResourceIds": ["azure monitor"]` — anchors the workbook to Azure Monitor context so it loads without needing a specific resource scope

## Deployment Options

There are two ways to deploy the workbook — Terraform (automated, repeatable) or manual import via the Azure Portal (quick, no tooling required).

---

## Option 1: Manual Import via Azure Portal

No tooling required. Use this for a quick one-off deployment.

1. In the Azure Portal, navigate to **Monitor** → **Workbooks**
2. Click **+ New**
3. Click the **Advanced Editor** button (`</>`) in the toolbar
4. Delete the default content and paste the full contents of [`workbook/arc-dashboard.json`](workbook/arc-dashboard.json)
5. Click **Apply**
6. Click **Save** and fill in:
   - **Title** — e.g. `Arc Servers Dashboard`
   - **Subscription** / **Resource group** — where you want to save it
   - **Location**
   - Set visibility to **Shared** so others in your organization can access it
7. Click **Apply**

The workbook is now accessible under **Monitor** → **Workbooks** for anyone with Reader access.

## Option 2: Terraform Deployment

## Prerequisites

- [Terraform](https://developer.hashicorp.com/terraform/install) >= 1.3.0
- Azure CLI authenticated (`az login`) or a service principal configured via environment variables
- Contributor access on the target subscription

---

## Deployment

### 1. Clone and configure

```powershell
git clone <repo-url>
cd ArcDashboard
Copy-Item terraform.tfvars.example terraform.tfvars
```

Edit `terraform.tfvars`:

```hcl
resource_group_name   = "rg-arc-dashboard"
location              = "eastus"
workbook_display_name = "Arc Servers Dashboard"

tags = {
  environment = "production"
  workload    = "arc-dashboard"
  owner       = "your-team"
}
```

### 2. Initialize

```powershell
terraform init
```

Downloads the AzureRM and Random providers into `.terraform/`.

### 3. Plan

```powershell
terraform plan
```

Review the resources that will be created:
- 1 resource group
- 1 Azure Monitor Workbook

### 4. Apply

```powershell
terraform apply
```

On completion, Terraform outputs:

```
resource_group_name = "rg-arc-dashboard"
workbook_resource_id = "/subscriptions/.../workbooks/..."
workbook_portal_url  = "https://portal.azure.com/#resource/..."
```

Open `workbook_portal_url` directly to view the deployed workbook.

### Updating

After editing `workbook/arc-dashboard.json`, re-run:

```powershell
terraform apply
```

Terraform detects the `data_json` change and updates the workbook in-place.

### Destroying

```powershell
terraform destroy
```

Removes the workbook and resource group.

## Permissions Required to View the Workbook

The workbook queries Azure Resource Graph across all subscriptions the viewer has access to. To see results, the viewer needs at minimum:

- `Reader` role on each subscription containing Arc machines
- No additional permissions are required — ARG queries respect the viewer's own RBAC scope
